# 道藏知识可视化 Demo 设计说明

**日期**：2026-03-31

## 目标

构建一个可部署到 GitHub Pages 的纯静态站点，用真实数据管线处理《道德经》81 章与《南华真经》内篇 7 篇，并为《南华真经》其余 26 篇预留扩展位。站点包含首页、概念星图、寓言图鉴和阅读器四个核心视图。

## 范围

- 《道德经》：81 章全量抓取、切分、编号、自动标注、索引、图谱与前端数据。
- 《南华真经》：33 篇全量抓取与编号，内篇 7 篇完成自动标注、寓言区块提取、索引、图谱与前端数据。
- IMA 知识库：作为辅助校验源，生成独立报告，不阻塞主流程。
- 发布方式：本地脚本生成 `docs/` 下所有发布资源，通过 `python3 -m http.server` 即可预览。

## 非目标

- 不引入构建工具、数据库或后端服务。
- 不追求一次性完成《南华真经》外篇和杂篇的深度实体标注。
- 不复用 `shiji-kb-main` 的大型历史包袱，只借鉴方法论与若干结构经验。

## 系统架构

系统采用“离线生成 + 静态消费”的双层结构：

1. `scripts/` 负责抓取、切分、标注、校验、建图和发布。
2. `archive/` 保存不可变底本。
3. `chapter_md/` 保存带 Purple Number 和实体标注的工作文本。
4. `kg/` 保存本体、实体索引、消歧表与关系。
5. `data/` 保存前端友好的 JSON。
6. `docs/` 保存所有静态页面与发布副本。

## 关键设计决策

### 1. 数据优先，前端只消费生成结果

前端不直接解析 `〖TYPE〗` 语法，只读取 `chapters.json`、`concept_graph.json`、`fable_graph.json` 等结构化 JSON。这样可以确保阅读器、高亮、索引和校验使用同一套解析规则。

### 2. 混合抓取策略

- 《道德经》使用 ctext API `gettext?urn=ctp:dao-de-jing`。
- 《南华真经》使用 ctext 网页抓取单篇页面，利用 `td.ctext` 提取正文。

这样既保持数据真实，又规避 `zhuangzi` API 认证限制。

### 3. 自动标注采用“词典优先 + 最长匹配”

第一版不做 LLM 在线标注，而使用项目内置词典与规则进行自动标注：

- 先按章节类型加载不同词典。
- 使用最长匹配避免短词打断长词。
- 始终在纯文本基础上插入标记，禁止嵌套标注。

这保证了可重复运行和可验证性。

### 4. 《南华真经》分层交付

- 所有 33 篇进入底本和段落编号层。
- 仅内篇 7 篇进入实体标注、寓言区块与图谱层。
- 前端对未标注章节仍可阅读，但不显示实体高亮或寓言网络。

## 数据模型

### `chapter_md/*.tagged.md`

- 标题行：`# 第一章` 或 `# 逍遙遊（內篇）`
- 段落行：`[1] ...`
- 寓言区块：`::: fable 名称`

### `entity_index.json`

- `metadata`：统计信息与生成时间
- `entities`：按实体 ID 存储名称、类型、别名、引用、频次、描述

### `chapters.json`

- `metadata`：书名、作者、章节数、字数、来源
- `chapters[]`：章节级数据
- `paragraphs[]`：段落文本与渲染片段

### `concept_graph.json`

- `nodes[]`：概念节点
- `links[]`：共现、对立、蕴含
- `categories`：类别到颜色和节点 ID 的映射

### `fable_graph.json`

- `fables[]`：寓言元数据
- `characters[]`：人物/角色聚合
- `themes[]`：主题聚合
- `links[]`：出场、辩论、主题关联

## 前端设计

### 首页

- 两张主卡片分别通向《道德经》概念星图与《南华真经》寓言图鉴。
- 底部提供阅读器入口和数据来源说明。
- 统计数字从 `site_stats.json` 动态加载。

### 概念星图

- D3 力导向图。
- 节点大小映射频次，颜色映射类别。
- 右侧详情面板显示引文、章节与跳读入口。

### 寓言图鉴

- D3 网络图，支持“寓言模式 / 人物模式”切换。
- 右侧显示寓言详情、主题、人物和跳读链接。
- 默认优先展示内篇 7 篇。

### 阅读器

- 按 URL 参数加载指定章节。
- PN 可复制链接。
- 实体类型开关可控制高亮显示。
- 支持上下章跳转与章节选择。

## 校验策略

### 强校验

- 去标记后文本必须与底本逐字一致。
- PN 编号必须从 1 连续递增。
- 不允许嵌套标注。
- 生成的图谱链接必须引用存在的章节或寓言。

### 弱校验

- IMA 搜索结果仅用于交叉比对，不作为真值源。
- 自动标注结果允许后续人工修订。

## 里程碑

1. 项目骨架、规范文件、测试骨架。
2. 真实抓取与切分。
3. 自动标注与强校验。
4. 实体索引、概念图谱、寓言图谱。
5. 静态前端与发布同步。

## 风险与缓解

- ctext 页面结构变化：抓取逻辑集中在一个模块，便于修复。
- 自动标注精度有限：使用 seed lexicon 和验证报告暴露问题。
- 前端与数据结构漂移：前端只消费脚本产物，不做二次解析。
